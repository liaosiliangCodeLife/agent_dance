# Agent Dance — 产品需求文档

> 版本: v2.0 | 日期: 2026-06-15

---

## 一、产品愿景

### 做什么

**Agent Dance** 是一个**去中心化、安全、隐私优先的智能体交互网络**。它给用户一个集中、私有的智能体管理中心——自己的入口，自己的调度，自己的数据，不依赖任何第三方 IM 平台。

### 为什么做

现在的智能体产品（OpenClaw、Hermes、Claude Code 等）远程交互全部走 IM 通道（微信、企微、Telegram、Discord）。这带来三个致命问题：

| 问题 | 影响 |
|------|------|
| **中心化依赖** | 消息路由、存储全在平台手里，随时可被切断 |
| **隐私风险** | 对话数据经过第三方服务器，敏感信息泄露 |
| **受制于人** | API 限制、封禁风险、审查机制不可控 |

用户和智能体之间的通信，不应该被任何中间平台控制。

### 目标

一个**端到端加密的智能体交互网络**，让用户与自己的智能体安全、自由地沟通。

---

## 二、产品定位

Agent Dance 是智能体领域的"个人服务器"——类比：

| 传统方案 | Agent Dance |
|---------|-------------|
| 微信 / Telegram / Discord 作为入口 | **用户自己**作为入口 |
| IM 平台存储对话历史 | **用户本地**私有存储 |
| 平台决定你能用什么智能体 | **用户自主**选择和管理智能体 |
| 消息经过平台服务器 | **端到端加密**，直连通信 |

---

## 三、目标用户

### 核心用户

- **AI 重度用户**：日常依赖多个 AI 智能体完成工作（编程、写作、信息检索）
- **隐私敏感者**：不希望对话内容被第三方平台收集和分析
- **自建智能体用户**：运行了自己的 Hermes / Claude Code 等智能体，需要一个统一的远程交互入口

### 使用场景

| 场景 | 说明 |
|------|------|
| 移动端远程操控 | 出门在外，通过手机操控家里的电脑上的智能体执行任务 |
| 多智能体协作 | 同时管理多个智能体，按需调度（编码 agent、写作 agent、搜索 agent） |
| 隐私对话 | 敏感话题、商业机密、个人隐私的 AI 对话不走第三方服务器 |
| 团队共享 | 安全地向团队成员授权使用特定智能体，不经过公有云 |

## 四、核心功能

### 4.1 智能体管理

| 功能 | 说明 |
|------|------|
| 添加智能体 | 通过地址/密钥添加智能体节点（Hermes、Claude Code、自定义 Agent） |
| 智能体列表 | 展示所有已添加的智能体，含在线状态、能力标签 |
| 智能体配置 | 修改名称、调整模型参数、设置访问权限 |
| 智能体发现 | 局域网自动发现附近的智能体节点 |
| 健康监控 | 实时显示智能体运行状态、资源占用 |

### 4.2 端到端加密通信

- 用户与智能体之间**直连通信**，不经第三方服务器
- 支持**局域网直连**（同 WiFi）和**远程穿透**（NAT 穿越 / 中继）
- 所有消息**端到端加密**，中继节点无法解密
- 连接方式：P2P（WebRTC / QUIC）+ 隧道（用户自建穿透服务器）

### 4.3 对话交互

> **核心约束：服务器级单轮对话** — 每个服务器对应一个对话入口，点击进入。每次发送消息不携带任何历史上下文，每次都是全新对话。API 请求的 `messages` 数组只包含当前一条用户消息。

#### 文字对话
- 流式逐字返回（打字机效果）
- 支持 Markdown 富文本渲染（代码块、表格、列表、公式）
- 链接安全预览

#### 思考过程展示
- DeepSeek 等推理模型的思考过程实时展示
- 思考过程可折叠/展开，视觉上与正式回答区分

#### 图片输入
- 拍照 / 相册选择，自动压缩后发送
- 图片缩略图在对话气泡中展示，可点击全屏预览

#### 语音输入
- 长按录音，松开发送
- 录音中显示波形动画 + 时长计数 + "上滑取消"
- 语音在智能体端 STT 引擎转录（faster-whisper 等）

#### 工具进度
- AI 调用工具时（搜索、代码执行），显示实时进度条和状态文字

### 4.4 对话约束

| 约束 | 说明 |
|------|------|
| 单轮对话 | API 请求不携带消息历史，每次只传当前一条 user message |
| 无会话概念 | 不保存历史对话、无多会话、无自动标题 |
| 服务器级入口 | 每个服务器一个对话入口，切换服务器即切换对话上下文 |
| 临时展示 | 对话页内容仅在当前展示时保留，退出即清空 |

### 4.5 隐私与安全

| 层级 | 措施 |
|------|------|
| 传输 | 端到端加密（TLS + 应用层加密） |
| 存储 | 本地加密数据库（SQLCipher） |
| 认证 | 公私钥对认证，无需密码传输 |
| 授权 | 用户可对每个智能体设置访问权限，支持临时授权码分享 |

### 4.6 智能体指令审批（Runs API）

当智能体检测到危险操作需用户批准时，App 弹出确认框：
- 显示即将执行的命令和风险说明
- 选项：允许本次 / 本次会话 / 始终允许 / 拒绝
- 60 秒无操作自动批准（防卡死）

## 五、通信架构

### 连接模式

```
┌──────────────┐         端到端加密         ┌──────────────┐
│   Agent Dance │ ◀══════════════════════▶ │   智能体节点   │
│  (移动端App)  │       P2P / 隧道          │ (PC / 服务器) │
└──────────────┘                           └──────────────┘
       │
       │ 局域网直连 (mDNS/LAN)
       │ 或 远程穿透 (NAT Traversal / 中继)
       │
       ▼
┌──────────────┐
│  用户自建隧道  │  ← 用户完全掌控，不经过第三方 IM
│  (NPS/frp等)  │
└──────────────┘
```

### 不依赖的组件

- ❌ 微信 / 企微
- ❌ Telegram / Discord
- ❌ Slack / 钉钉 / 飞书
- ❌ 任何公有云 IM 服务

## 六、UI/UX 设计

### 6.1 页面结构

```
首页：智能体列表
├── 智能体卡片（名称 + 在线状态 + 能力标签）
├── 点击进入对话
└── 下拉刷新在线状态

对话页
├── 顶部：智能体名称 + 当前会话
├── 消息列表
│   ├── 用户气泡（右对齐）
│   └── AI 气泡（左对齐，含思考区 + Markdown 正文）
├── 工具进度条
└── 底部输入栏（附件 + 文字 + 语音 + 发送）

智能体管理页
├── 添加智能体（地址 + 密钥）
├── 已添加列表（可编辑/删除）
└── 网络状态诊断

设置页
├── 主题（亮色 / 暗色 / 跟随系统）
├── 网络配置（隧道地址、中继设置）
├── 安全（密钥管理、生物识别锁定）
└── 数据管理（导出、清除）
```

### 6.2 消息气泡

| 属性 | 用户消息 | AI 消息 |
|------|---------|---------|
| 对齐 | 靠右 | 靠左 |
| 背景色 | 主题主色 | 灰色容器色 |
| 圆角 | 16px | 16px |
| 内容 | 纯文本 + 图片缩略图 | Markdown |

### 6.3 思考过程区域

- 灰色小字号斜体 + 左侧竖线装饰
- 默认折叠，流式输出时展开
- 图标：💭

### 6.4 输入栏

```
┌─────┬──────────────────────┬─────┬──────┐
│ 📎  │ 多行输入 (1~5行)      │ 🎤  │ ✈️   │
└─────┴──────────────────────┴─────┴──────┘
```

- 📎：图片选择（拍照 / 相册）
- 🎤：长按录音
- ✈️：发送（生成中变为 ⏹ 停止）

### 6.5 主题与错误

- 主题跟随系统（亮色 / 暗色自动切换）
- 错误提示统一用 SnackBar

## 八、非功能性需求

| 类别 | 要求 |
|------|------|
| 平台 | Android 8.0+ / iOS 16.0+ |
| 加密 | 端到端（传输）+ 静态（本地存储） |
| 延迟 | 局域网直连 < 100ms，远程穿透 < 500ms |
| 可靠性 | P2P 失败自动降级为中继模式 |
| 离线 | 无网络时可浏览历史对话 |
| 数据主权 | 所有数据存储在用户设备，不上传任何第三方服务器 |

## 九、与现有方案对比

| 维度 | OpenClaw / Hermes + IM | Claude Code 等 CLI | **Agent Dance** |
|------|----------------------|-------------------|-----------------|
| 入口 | 微信/企微/TG/DC | 终端命令行 | **独立 App** |
| 通信 | 经过 IM 服务器 | SSH/本地 | **P2P 直连加密** |
| 隐私 | 经过第三方 | ✅ | **✅ 更强** |
| 移动端 | ✅（借 IM） | ❌ | **✅ 原生** |
| 多智能体 | 需多个 Bot | — | **统一管理** |
| 去中心化 | ❌ | ✅ | **✅** |
| 用户体验 | 受限于 IM | 高门槛 | **原生流畅** |
