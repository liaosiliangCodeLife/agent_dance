# Agent Dance Release Notes

---

## v3.2 — 服务器→会话→对话三级结构 (2026-06-24)

### 架构重构

```
v3.1:  服务器列表 → 直接进聊天（一个服务器 = 一个对话）
v3.2:  服务器列表 → 会话列表 → 聊天页（一个服务器 = N 个独立会话）
```

点击服务器不再直接进聊天，而是先进会话列表页，每个服务器下可以创建多个独立会话。

### 新增功能

| 编号 | 功能 | 说明 |
|------|------|------|
| F-131 | 会话列表 | 服务器下所有会话，按活跃时间倒序，显示标题+最后消息预览 |
| F-132 | 新建会话 | 点 + 按钮自动生成标题，可后续改名 |
| F-133 | 删除会话 | 左滑删除，聊天记录同步清除 |
| F-134 | 修改标题 | 长按标题进入编辑 |
| F-135 | 独立 ID | 每个会话 UUID，对应服务端 `X-Hermes-Session-Id` |
| F-136 | 进入对话 | 点击会话加载历史消息 |
| F-137 | 快速切换 | 顶部标题点击下拉切换其他会话 |
| F-138 | 用户头像 | 对话页自己的气泡旁显示头像 |

### API 层改动

每次请求携带两个 header：

```
X-Hermes-Session-Id: {sessionId}           ← 会话隔离上下文
X-Hermes-Session-Key: agent_dance:{serverId} ← 长期记忆绑定
```

## v3.1 — 多轮上下文对话 (2026-06-24)

### 废除

| 编号 | 说明 |
|------|------|
| ~~F-126~~ | ~~单轮对话约束，每次请求只发当前消息~~ |

### 新增

| 编号 | 功能 | 说明 |
|------|------|------|
| F-129 | 多轮上下文 | 每次发送携带完整对话历史，AI 知道之前聊过什么 |
| F-130 | 会话持久化 | 切换服务器/退出 App 再回来，历史从 SQLite 恢复 |

### 实现方案

客户端拼装完整 `messages` 数组发给 API，不再只发当前一条消息。新增 `chat_api_messages.dart` 专门处理消息格式转换。
